# 插件论文解读模式快速开始

当前模式无需启动后端服务，插件会先在本地完成 PDF 文本解析，再调用你配置的模型 API 生成中文论文解读 Markdown。

## 1. 在 Chrome 中加载插件
1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择目录：`paper_pdf_flow_skill/extension`

## 2. 使用流程
1. 打开任意网页，点击浏览器扩展按钮
2. 进入 `API配置`，填写地址、API 密钥、模型名字，并先测试连接
3. 返回主页面，选择一个 PDF 文件
4. 可选填写输出文件名（必须以 `.md` 结尾）
5. 点击“开始生成”
6. 等待自动下载 Markdown 结果（默认输出为中文论文解读稿）

## 3. 当前默认输出结构
- `# <pdf_stem> 论文解读`
- `## 文献信息`
- `## 这篇论文要解决什么问题`
- `## 按原文结构解读全文`
- `## Method 详细解读`
- `## 实验设计、数据与评价指标`
- `## 结果如何支撑结论`
- `## 局限性与启发`
- `## 一句话总结`

## 4. 面板行为（最小开关规则）
- 扩展按钮点击行为：
  - 面板关闭时：打开并重置
  - 面板打开时：关闭
- 失焦、切换标签、切桌面、点击页面其他位置：保持当前状态，不自动开关。
- 页面刷新后默认关闭（不自动恢复）。

## 说明
- PDF 解析仍在网页内常驻面板本地执行。
- 当前输出目标为“全文解读 + Method 细讲”。
- 当前解析引擎为打包到插件内的 `pdf.js` + 启发式文本规则，模型端再基于本地解析结果完成结构化解读。
