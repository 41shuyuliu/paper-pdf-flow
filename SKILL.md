# 论文 PDF 解读技能

使用本技能可将单篇论文 PDF 转换为结构化中文“论文解读”Markdown，重点是帮助用户真正读懂论文，而不是只生成按图摘要。

默认输出结构如下：
- `# <pdf_stem> 论文解读`
- `## 文献信息`
- `## 这篇论文要解决什么问题`
- `## 按原文结构解读全文`
- `## Method 详细解读`
- `## 实验设计、数据与评价指标`
- `## 结果如何支撑结论`
- `## 局限性与启发`
- `## 一句话总结`

## 适用场景

- 用户希望快速读懂整篇论文，而不只是看图得出结论。
- 用户特别关注 `method`、实验步骤、技术细节与执行顺序。
- 需要面向非专业背景读者输出中文解读。
- 需要保留单文件 Markdown 产物，便于项目记录与复用。

## 执行流程

1. 校验输入 PDF 是否存在，并优先提取正文、方法、结果与图注信息。
2. 以原文结构为框架生成中文解读稿，`Method` 部分必须是全文最详细的部分。
3. 允许参考图注和辅助提取结果，但最终输出必须忠实于原文。
4. 如信息不明确，明确写“未自动识别（建议手动补充）”或“原文摘录中未明确给出（建议回看原文）”。
5. 仅输出到 `--out` 指定路径。

## 解读要求

- 用中文作答，风格应像资深研究员在白板前带学生读论文：简洁、清楚、便于理解。
- 工作目标不是写一篇长摘要，而是帮助用户按论文原有逻辑读懂全文。
- 允许添加补充解释帮助理解，但不得编造原文未提供的事实、实验步骤、数据或结论。
- 优先依据正文内容解读，再用图注补强结果证据链。
- `Method` 部分要逐步、逐点解释研究对象、输入输出、关键模块、执行顺序、训练或实验设置、评价方式及其关系。
- `实验设计、数据与评价指标` 部分应尽可能交代数据来源、分组、对照、时间点、剂量、指标与比较方式；若材料不足，必须明确说明。

## 单文件输出约束（严格）

- `--out` 是唯一输出目标。
- 再次生成时覆盖 `--out`。
- 禁止创建任何副本或镜像文件，包括：
  - `README.md` 镜像文件，
  - `*_backup.md`，
  - `*_en.md`，
  - `*_copy.md`，
  - `同步副本` 或其他重复笔记。
- 如需翻译或重写，也必须复用同一 `--out` 路径覆盖写入。

## 本地辅助草稿（可选）

仓库内的 `scripts/pdf_to_flow_note.py` 仍可作为本地辅助提取工具使用，但当前更适合作为“辅助提取报告 / 原始抽取 payload 生成器”，而不是新的默认最终产物生成器。

```powershell
python .\scripts\pdf_to_flow_note.py `
  --pdf "path\to\paper.pdf" `
  --out "path\to\README.md" `
  --lang zh `
  --format markdown
```

使用建议：
- 当你需要快速获得标题、DOI、图注、实验信号、正文摘录等辅助信息时，可先运行该脚本。
- 若你想把抽取结果交给其他程序处理，可改用 `--format json` 输出原始 payload。
- 正式产物应以结构化论文解读稿为准；该脚本不再承担最终稿生成职责。

## 输出要求

生成结果至少包含：
- 文献元信息（标题/期刊线索/DOI，若可识别）
- 问题定义
- 按原文结构的全文解读
- 详细的 `Method` 拆解
- 实验设计、数据与评价指标
- 结果与结论之间的证据链说明
- 局限性与启发
- 单文件输出（禁止副本）

对于排版复杂、OCR 质量差或噪声较高的 PDF，建议在生成后做一次快速事实核对。
