# 架构设计（MVP）

## 模块划分
- `core/`：共享的 PDF 提取、prompt 构建、模型请求与辅助提取报告逻辑。
- `backend/`：API 服务（上传 PDF、触发论文解读、查询任务状态、下载最终 Markdown）。
- `extension/`：浏览器插件界面（本地解析 PDF、直连模型、自动下载最终解读稿）。
- `scripts/`：本地辅助提取入口，仅输出辅助提取报告或原始抽取 payload，不再生成最终论文解读稿。
- `docs/`：设计文档与接入说明。

## 当前链路
1. 插件链路：本地解析 PDF -> 组装固定 prompt -> 调模型 -> 下载最终论文解读稿。
2. 后端链路：上传 PDF -> `core/` 提取正文/图注/信号 -> 组装同一套 prompt -> 调模型 -> 下载最终论文解读稿。
3. 脚本链路：读取 PDF -> 输出辅助提取报告或 JSON payload，供人工检查或其他链路复用。

## 设计原则
- 全项目只保留一个正式产物契约：结构化“论文解读”Markdown。
- 插件与后端共享同一套输出结构，不再允许后端走旧的“按图流程笔记”生成路径。
- 辅助脚本只负责提取，不承担最终稿生成职责。
